What the engine does

areadetail
Libraryall 37 Zotero reference types (base-field renames resolved), standalone + child notes, nested collections, colored tags, related items, trash + restore, group libraries
AcquireDOI (CrossRef) · ISBN (Open Library) · PMID (NCBI) · arXiv; retrieve-metadata-from-PDF; import BibTeX / RIS / CSL-JSON / EndNote XML / MODS / MARCXML / Zotero RDF; Zotero account pull (Web API) + WebDAV attachment sync
CiteAPA, MLA, Chicago (author-date + notes-bibliography), IEEE — or any loaded .csl via the built-in CSL processor; BibTeX cite-key generation
Findquick search across every field (incl. indexed text/HTML/PDF attachments), condition-based saved searches, duplicate detection + merge
ExportBibTeX, RIS, CSL-JSON, EndNote XML, MODS, MARCXML, Zotero RDF — by selection, collection, or whole library

Build model

The engine builds as rlib + staticlib + cdylib (cargo build in crates/zcite-core) with a unit + integration test suite, including a --no-default-features pure-core build for headless CI. The same engine embeds inside the other MenkeTechnologies GUI apps.
