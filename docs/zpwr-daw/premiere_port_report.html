// Every Adobe Premiere Pro feature, mapped to where it lives in zpwr-daw's video editor — a citation-backed coverage audit and roadmap.
// The video editor rides the shared clip engine (libs/zpwr-clip-engine): the same timeline/clip arranger that drives MIDI and audio now carries video clips — a video track type, in-browser filmstrip thumbnails, in/out trim, and a program monitor that previews the playhead frame and pops out into a draggable player. Premiere's deep surface — effects & keyframing, Lumetri color, transitions, Essential Graphics, audio mixing, multicam, Media Encoder export — is the roadmap below, built area by area.

⚠ Read this first — what this report is, and isn't

This is a citation-backed coverage map and roadmap: every PORTED row names the exact file:line (or file + function) in this repo where the feature lives — the citation is the proof, not a self-assigned score. There is no ‘100% / 0 gaps’ claim — the opposite: most of Premiere is GAP (real, in-scope, not yet built). What's PORTED today is the video-clip foundation (Phase 1+2): clips on the timeline, filmstrip, in/out trim, the program monitor + pop-out player, and the multi-track timeline editing inherited from the DAW arranger.
