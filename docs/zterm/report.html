zterm is a proprietary fork of Alacritty v0.17.0, hardened and rebranded as a first-party MenkeTechnologies component intended to fuse with the zshrs runtime. This page summarizes the fork model and feature surface as it stands.

Fork model

aspectdetail
upstreamAlacritty v0.17.0
statusproprietary first-party fork, rebranded as zterm
platformsLinux, BSD, macOS, Windows
buildcargo build → target/debug/zterm; vendored, self-contained, no external runtime deps
directionintended to fuse with the zshrs runtime

Feature surface

areadetail
RendererOpenGL ES 2.0 glyph-atlas renderer
VT parsingfull xterm-compatible (vim, tmux, htop, ncurses)
Color24-bit truecolor
Navigationscrollback, vi mode, search, hints
terminfosets TERM=zterm when installed (else xterm-256color); source in extra/zterm.info

Configuration

orderpath
1$XDG_CONFIG_HOME/zterm/zterm.toml
2$XDG_CONFIG_HOME/zterm.toml
3$HOME/.config/zterm/zterm.toml
4$HOME/.zterm.toml
5/etc/zterm/zterm.toml (Windows: %APPDATA%\zterm\zterm.toml)

No config file is created for you; first match wins. Full configuration is documented in man 5 zterm. A feature overview also ships in docs/features.md in the source tree.
