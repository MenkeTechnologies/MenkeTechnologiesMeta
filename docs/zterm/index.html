ZTERM // GPU-ACCELERATED TERMINAL EMULATOR

// Alacritty v0.17.0 fork · OpenGL ES 2.0 · Linux · BSD · macOS · Windows
Engineering Report Source

zterm is a fast, GPU-accelerated, cross-platform terminal emulator and part of the MenkeTechnologies stack. It is a proprietary fork of Alacritty v0.17.0, hardened and rebranded as a first-party component intended to fuse with the zshrs runtime. Supported platforms: Linux, BSD, macOS, and Windows.

GPU Renderer

An OpenGL ES 2.0 glyph-atlas renderer for high-throughput, low-latency drawing.

Full VT Parsing

xterm-compatible VT parsing — vim, tmux, htop and ncurses apps render correctly.

Modern Terminal

24-bit truecolor, scrollback, vi mode, search, and hints.

Self-Contained

A vendored build with no external runtime dependencies; ships its own zterm terminfo.

Configuration

zterm does not create a config file for you. It looks for one in order: $XDG_CONFIG_HOME/zterm/zterm.toml, $XDG_CONFIG_HOME/zterm.toml, $HOME/.config/zterm/zterm.toml, $HOME/.zterm.toml, then /etc/zterm/zterm.toml (on Windows: %APPDATA%\zterm\zterm.toml). The full configuration is documented in man 5 zterm. It sets TERM to zterm when that terminfo entry is installed, otherwise falls back to xterm-256color; the terminfo source ships in extra/zterm.info.

Build

A debug build is cargo build, producing the binary at target/debug/zterm. See the engineering report for the fork model and feature surface, and the MenkeTechnologiesMeta umbrella for the wider stack.
