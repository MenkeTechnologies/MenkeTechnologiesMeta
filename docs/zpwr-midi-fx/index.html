>_JUCE MIDI-EFFECTS PLUGIN

The MIDI-domain companion to zpwr-fx. The same fully modular patch-graph engine, instantiated on the note stream. Paid product.

Concept

Where zpwr-fx processes the audio stream, zpwr-midi-fx transforms the MIDI stream before it reaches an instrument — arpeggiation, chord generation, scale quantization, Euclidean / generative sequencing, velocity and timing humanize, and note remapping. It is the same free-routed zpwr-patch-core graph as zpwr-fx, instantiated on zmidi::NoteStream instead of float — 56 note-stream module types, not a fixed-slot rack.

Module library

56+ note-stream module types (registerMidiModules), grouped by family. Any block wires to any block; each has note-stream inputs, a scalar Mod input, up to six parameters, and one output.

Note-offs are scheduled across audio-block boundaries, so nothing ever hangs. The graph evaluates once per block in topological order. Full per-module reference: reference.html.

Mod matrix & cables

A dynamic route list maps a scalar source → any block parameter with a depth. Sources: an expandable pool of host-automatable soft keys (8 active by default), LFO / Env block outputs, Random, performance controllers (mod wheel, pitch bend, aftertouch, velocity, expression CC11, sustain CC64), and MPE per-note bend / pressure / slide. The same sources are patchable into each block's Mod input.

Cables drag to re-patch; right-click for a per-cable Level (scales note velocity on that connection — 0 mutes), Colour, and Disconnect. Routing rebuilds the graph; depth/level are live lock-free tweaks. The ⚡ EZ WIRE button auto-wires MIDI In → blocks → Out for a working chain without touching cables.

Presets

A factory bank ships with the plugin (Chord Arp, Strummed Chords, Euclidean Pulse, Fifth Harmonizer, Arp Echo). User patches save as *.zmfxpatch under the user application-data directory and are managed from the PRESETS tab. A patch — blocks, routing, mods — is zpwr-patch-core JSON the WebView edits and the plugin persists in its state.
