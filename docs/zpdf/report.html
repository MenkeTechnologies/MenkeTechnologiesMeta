Architecture

zpdf is the GUI (Tauri v2 desktop app). zpdf-core is the embeddable engine/logic it consumes as a submodule — pure Rust, backed by lopdf (MIT), no GUI or platform deps. The split is what lets the engine drop into other hosts (traderview) while the desktop app is just one front end. The engine is real and tested; the Tauri shell is in progress.

Coverage

The port report catalogs the full Adobe-Acrobat / macOS-Preview surface and derives each feature's status from the zpdf-core source — a feature is marked done only when a real, non-stub engine symbol exists. The headline numbers above come from that audit; open the report for the per-feature, per-category breakdown.

Engine verification

zpdf-core ships integration tests that build valid PDFs in memory (no external fixtures): page counting, a Unicode metadata save/reopen round-trip, rotation normalization and out-of-range handling, annotation persistence across save/load, and confirmation that every unimplemented capability returns a typed error rather than panicking. CI-safe on headless Linux.
