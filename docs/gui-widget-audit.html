>_Executive summary

The house rule is GUI-only zgui components: every app builds its UI from the shared zgui-core library (ZGui.*) — modals, dropdowns, pickers, tables, tooltips — never a hand-rolled or native browser widget. This audit greps each app's own frontend for the native form-control elements that a ZGui.* component already covers, so they read as concrete migration targets, not vibes. Most apps are clean or near-clean; the outliers are traderview and Audio-Haxor, whose forms lean heavily on native <select> / <input type=range|color>.

~Detection method

Verifiable, not heuristic. For each app's own frontend (frontend/**/*.{js,html}, excluding the vendored lib/zgui-core, vendor/, node_modules, bundles and dist) each native widget element is counted with a fixed grep, and each maps to the ZGui.* component it should be. The JS dialog metric (alert/confirm/prompt) was deliberately excluded — apps wrap those in a ZGui.modal.prompt-delegating helper, so counting the calls (or the window.* fallback inside the helper) mislabels correct ZGui usage as a violation.

*Findings

  • traderview is the primary migration target. 574 native <select> across 404 files plus 221 native date/time inputs, against only ~14 ZGui.* component calls — its forms (setup wizard, currencies, business-context, clip presets) are almost entirely native. Highest payoff: route selects through ZGui.dropdownMenu and dates through ZGui.datePicker.
  • Audio-Haxor is the second offender — 63 native range sliders (volume faders) + 17 color inputs + 65 selects. The faders should be ZGui.dbFader / slider and the colour inputs ZGui.colorPicker, both already used elsewhere in the fleet.
  • Four apps are already fully clean (0 native widgets, all ZGui): zoffice, zthrottle, zcontainer, zphoto. zcontainer (605 ZGui calls) and zthrottle (114) are the reference implementations for how the rest should look.
  • The rest are minor — a handful of native <select> / one-off colour pickers each (2–13 elements), a bounded cleanup. zemail (196 ZGui calls) and zftp (214) are near-clean already.
  • No native <dialog> anywhere — modals are uniformly ZGui.modal across the fleet, and the alert/confirm/prompt calls all route through a ZGui.modal.prompt-delegating helper (which is why that metric was excluded rather than counted as a violation).
