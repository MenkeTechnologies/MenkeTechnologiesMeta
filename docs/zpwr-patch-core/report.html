Runtime model

Nodes are wired by source id with fan-out and feedback. The runtime graph is rebuilt in topological order on each structural edit; cycles resolve with a one-sample delay. Every node param carries a (source, depth) mod-matrix entry, and every cable has its own gain + colour. Live param edits are lock-free (atomic params); structural edits build a new graph and swap it atomically so the audio thread never blocks.

g->evalSample (timeIndex, ext, N);
out = g->sourceValue (g->outputSource (0)) * g->outputGain (0);

Each node exposes an S signal output plus a float scalar projection used as its mod-matrix value; the float instantiation is the plain audio graph.

Serialisation & migration

patchToJson stamps "v": kPatchJsonVersion; patchJsonVersion(json) reads it (missing ⇒ 1 = legacy). migrateSourceIds(patch, remap) rewrites every source id a patch references — input cables, mod sources, outputs — so hosts can shift a legacy external-source layout forward on load.
