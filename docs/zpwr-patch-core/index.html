>_SIGNAL-AGNOSTIC PATCH GRAPH

The shared cable-routing core behind the plugin stack. Knows nothing about audio or MIDI. Private core.

Concept

It owns the parts that are identical in every modular plugin and nothing else, so zpwr-fx, zpwr-synth, and zpwr-midi-fx share one routing engine instead of three copies:

  • Routing — nodes wired by source ids, fan-out, feedback.
  • Evaluation — topological order each rebuild; cycles resolve with a one-sample delay.
  • Mod matrix — every node param has a (source, depth) modulation.
  • Per-cable gain + colour.
  • Lock-free live edits (atomic params) + atomic graph swap on structural edits.
  • JSON serialisation of a whole patch, with versioning + source-id migration.
  • ScriptEngine — a small RT-safe expression VM, exposed as the Expr module.

Signal-agnostic by design

The graph is templated on the signal type carried between nodes (a SignalTraits<S> policy): float for audio (zpwr-fx, zpwr-synth) and a note-event stream for MIDI (zpwr-midi-fx). Each host supplies a ModuleRegistry of node types and the external source values per sample; the core resolves block outputs and leaves the rest to the host.

Shared WebEditor

zpc::WebEditor<Engine> is the WebView backend every host shares — catalog / patch JSON, BinaryData serving, preset I/O, ~30 native functions. Soft knobs are an expandable pool with a fixed automatable ceiling; the UI's +/− controls grow the active count without shifting host source ids. EZ mode exposes an ⚡ EZ WIRE button that connects the standard signal path for linear hosts.

Dependency

C++20juce_core onlyheader + lib

Consumers add it via add_subdirectory and link zpwr::patch_core. No audio, MIDI, GUI, or networking coupling — JUCE's curl path is disabled.

Module registry

The audio/synth DSP pack ships here too (zpc::buildFxRegistry / registerAudioModules), so zpwr-fx and zpwr-synth share one catalog: 900+ float module types across dynamics, EQ/filter, delay, reverb, modulation, distortion/saturation, pitch, spectral (FFT), stereo, lo-fi, and creative/glitch families — including a 100+-model analog pack (registerAnalog: classic synth filters + 1176/LA-2A/Fairchild/dbx/ SSL/Distressor compressors, Pultec/Neve/API/SSL/Manley EQs, Neve/API/tube preamps + Studer/Ampex tape, Tube Screamer/RAT/Big Muff/Klon pedals, and Fender/Marshall/Vox/Mesa amps — all generic topologies). The MIDI instantiation adds 66 note-stream modules in zpwr-midi-fx — 1000+ DSP blocks across the stack. Effect-by-effect parity against every major DAW + plugin catalog — and against the Bitwig Grid — is tracked in FX_PARITY.md (synth/oscillator parity in SYNTH_PARITY.md).

Defining a module

A node type is a ModuleInfo: name, one-line description, category, params (min/max/default), input count, and a compute lambda over a ComputeContext. description + category are the source of truth for the generated reference docs:

zpc::ModuleInfo m;
m.name = "Gain";
m.description = "Gain trim (dB) plus a DC bias offset.";
m.category = "Utility";
m.params = { { "Gain", -60, 24, 0 } };
m.numIns = 1;
m.compute = [] (const zpc::ComputeContext& c) {
    return c.in[0] * std::pow (10.0f, c.params[0] * 0.05f);
};
reg.add (std::move (m));

String literals must be ASCII (they load into juce::String via the const char* ctor); a CI linter enforces it.

Editor & host capabilities

Beyond catalog/patch/preset bridging, zpc::WebEditor and its companion headers give every host the same feature set for free:

  • BROWSE tab — SynthMaster-style faceted preset browser (PRODUCT / BANK / AUTHOR / TYPE / ATTRIBUTES / STYLES columns, multi-select AND-across / OR-within), fuzzy search, favourites (favorites.json), random-preset.
  • Wavetable editor — draw single-cycle frames on a canvas; Position morphs; Serum-style built-in tables.
  • Graphical LFO / envelope editors — draggable SVG ADSR + waveform panel with exact DSP shape math and a live playhead.
  • Global modulators (GlobalMods) — 3 LFOs, 2 envelopes, Random, S&H + the full MIDI/MPE source set (mod wheel, bend, aftertouch, velocity, expression, breath, sustain, MPE pressure/slide/bend, 8 assignable CCs), all 0..1.
  • Layers + bus routing (LayeredEngine) — unlimited layers (each a full engine copy), per-layer parallel/series route, mute/solo/gain/pan, full JSON round-trip.
  • Microtuning — Scala .scl → per-note cents table applied as a fractional Note external (every oscillator inherits it).
  • Colour-scheme editor, per-host log file, on-screen PERFORM keyboard with pitch-bend / mod wheels.

Versioning & migration

patchToJson stamps "v": kPatchJsonVersion; patchJsonVersion(json) reads it (missing ⇒ 1 = legacy). migrateSourceIds(patch, remap) rewrites every source id a patch references (input cables, mod sources, outputs), so a host can shift a legacy external-source layout forward on load — old patches that worked yesterday keep working.
