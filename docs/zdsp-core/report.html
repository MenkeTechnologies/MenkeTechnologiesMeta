zdsp-core is a header-only C++ shared component — JUCE-based DSP headers, no separate library artifact and no build step of its own. It is consumed by add_subdirectory(zdsp-core) + linking the zdsp::core interface target; JUCE (juce_audio_basics + juce_core) must already be present in the consuming project. This page describes the unit surface, where each unit came from, and the migration roadmap. Every unit is the generic kernel of something the Audio-Haxor audio-engine had already shipped inline.

Consumption model

A consuming app adds zdsp-core as a submodule (typically under libs/), calls add_subdirectory(libs/zdsp-core), and links zdsp::core. Because the library is header-only and links the host's own JUCE targets, there is no version skew between the app's JUCE and the DSP code. The units carry no host-specific assumptions beyond JUCE's audio-basics — they run identically inside Audio-Haxor's Tauri host and the JUCE plugin WebViews.
