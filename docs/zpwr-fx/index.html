>_JUCE MODULAR MULTI-EFFECTS

A fully modular patch graph of audio effects — free cabling, not a fixed slot rack. Paid product.

Architecture

zpwr-fx is the audio host for the shared zpwr-patch-core engine: a free-routed graph of modules (cables by source id, fan-out, feedback resolved with a one-sample delay) rather than a fixed slot chain. Every node param carries a (source, depth) entry in the mod matrix; live edits are lock-free with an atomic graph swap on structural changes; whole patches serialise to JSON with versioning + source-id migration.

Each layer is a full engine copy (layers are unlimited); the ⚡ EZ WIRE mode auto-chains In → blocks → Out so linear hosts never touch a cable. The WebView UI and its expandable soft-knob pool come from zpc::WebEditor.

Module catalog

900+ audio/synth module types in the shared registry (buildFxRegistry / registerAudioModules) — dynamics, EQ/filter, delay, reverb, modulation, distortion/saturation, pitch, spectral (FFT), stereo, lo-fi, and creative/glitch families (plus 66 note-stream modules in zpwr-midi-fx — 1000+ DSP blocks across the stack). Effect-by-effect parity against every major DAW + plugin catalog is tracked in FX_PARITY.md.

Build your own — primitives + code

Not just presets: wire the 900+ modules into your own algorithm (any node to any node, feedback, mod matrix), or write per-sample DSP in the Expr code module — a sandboxed VM with 8 automatable knobs, persistent state, ~30 math functions, and a fractional tap() delay line for hand-built filters/combs/delays. UAD and Eventide native plugins ship fixed emulations; the Bitwig Grid has primitives but no scripting node — zpwr-fx has both.

The Expr code module

Expr blocks run a user-written per-sample expression compiled by an in-house RT-safe VM — they do what the fixed blocks can't, edited live in the block's detail panel:

  • Vars — in, t, sr, p0..p7 (p0–p5 knobs, p6 = In 2, p7 = Mod), s0..s3 (persistent state), pi tau e.
  • Funcs — sin cos tan tanh floor frac wrap saw sqr tri min max pow fmod clamp lerp if step noise rand, and tap(d) — the block's own output d samples ago (fractional) for combs / Karplus / feedback.
  • Safe by construction — no loops, allocation, or memory access in the hot path; output sanitised (NaN/Inf → 0); over-limit programs fail to compile.

Example wavefolder: out = sin(in * (1 + p0 * 8) * pi). UAD and Eventide native plugins ship fixed emulations; the Bitwig Grid has primitives but no scripting node — zpwr-fx has both.

Mod matrix, presets & UI

Every node param carries a (source, depth) mod-matrix entry. Sources include the expandable soft-key macros (host-automatable, 8 active by default), any LFO/Envelope/Chaos output, and per-cable gain + colour. Live param edits are lock-free; structural edits build a new graph and swap it atomically so the audio thread never blocks.

The UI is a self-contained JUCE 8 WebView (cyberpunk neon theme) with Patch (sources / blocks / outputs grid + SVG cables + ⚡ EZ WIRE + per-block detail/code panel), Presets (factory + user browser), and Settings tabs. Factory presets (Stereo Slap, Filter Sweep, Comb Resonator, Wavefolder); user patches save as *.zfxpatch. Full plugin state round-trips through JUCE get/setStateInformation.
